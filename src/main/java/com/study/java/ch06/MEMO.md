## 객체 스트림
- 객체를 전송할 수 있는 방법

### 객체 전송 순서
1. 마샬링: 데이터를 특정 통신 채널로 보낼 수 있는 형태로 변환
2. 전송: 변환된 데이터를 전송/기록
3. 언마샬링: 변환된 데이터를 읽어서 원래 형태로 변환

#### 마샬링
- 데이터를 바이트 흐름으로 만들어 TCP와 같은 '통신 채널'을 통해 전송하거나 스트림으로 써 줄 수 있는 형태로 바꾸는 과정
- 마샬링 가능한 데이터
  - 기본 자료형: boolean, byte, char, short, int, long, float, double
  - Serialize 구현체
- 마샬링은 ObjectOutputStream.writeObject() 에 의해 직렬화(serialization)되어 전달
  - 객체 직렬화 == 객체를 '마샬링'

#### 전송
- 데이터를 발신지에서 목적질 전달하는 과정
- 전송을 위해 객체 스트림은 바이트 기반의 표준 스트림을 이용
- 네트워크에 대한 전송뿐만 아니라 파일에 쓰기를 할 경우에도 객체 스트림은 '바이트 기반'의 표준 스트림을 이용

#### 언마샬링
- 마샬링의 반대과정으로 '전송받은 데이터를 원래 형태로 변환하는 과정'
- 객체 직렬화의 반대 경우로 '역직렬화'라고 함
- ObjectInputStream.readObject() 에 의해 역직렬화(deserialization)

### 정리
- ObjectOutputStream 은 객체를 '직렬화' 하기 위해 사용
- ObjectInputStream 은 전송받은 데이터를 '역직렬화' 하기 위해 사용
